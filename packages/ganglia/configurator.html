Configuring ganglia

Ganglia should work out of the box but will be using the default settings. Here you can configure a few settings (mostly names) for your ganglia setup so that it will display the correct information about your nodes and/or cluster grid upon startup.

Name of Cluster
Name of Grid

Owner of Cluster (Department, Company, etc...)

Client nodes can have multiple network interfaces. Specify which interface should be used for sending/receiving ganglia monitoring information:

Sending Network Interface for ***Client*** nodes
Receiving Network Interface for ***Client*** nodes

Enable per-image ganglia gmond configuration

When multiple clusters are in use it might make sense to use different multicast ports for the client nodes such that the data from different clusters doesn't interfere. Specify which multicast port should be used for sending ganglia monitoring information:

Sending Network Port for ***Client*** nodes
Receiving Network Port for ***Client*** nodes
